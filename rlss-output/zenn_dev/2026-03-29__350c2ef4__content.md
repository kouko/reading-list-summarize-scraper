2026年3月にGhostty 1.3.0がリリースされました。新機能が盛りだくさんです。

リリース直後にいくつか問題が見つかり、日をおかずに1.3.1がリリースされています。これから使う方は1.3.1を導入しましょう。

この記事では個人的に気になった機能をピックアップして紹介します。筆者がmacOS環境なので、macOSで使えない機能に関しては触れていません。

前回の記事はこちら。

なお、以下で紹介しているキーバインドはあくまでmacOS版のデフォルトのキーです。すべて設定で変更可能なことと、別環境ではデフォルトが異なる場合があることを留意してください。

## 破壊的変更：クリップボードのコピー形式のデフォルト変更

最初に破壊的変更から。キーバインディングの `copy_to_clipboard` のデフォルト値が `mixed` に変更されました。

`mixed` はクリップボードに `text/plain` と `text/html` の両方をセットする形式です。これにより、コピー結果をリッチテキスト対応のエディタへペーストすると、色やスタイルが保持された状態で貼り付けられます。

この `copy_to_clipboard` にオプションが設定できるようになったのも1.3.0からで、以下のフォーマットを指定できるようになりました。

| フォーマット | 内容 |
| --- | --- |
| `mixed` | プレーンテキスト + HTML（新デフォルト） |
| `plain` | プレーンテキストのみ（従来の挙動） |
| `html` | HTMLのみ |
| `vt` | ターミナルエスケープシーケンスを保持 |

いちおう破壊的変更ではあるのですが、リッチテキストが効く場所でのみ関係するので、それほど困ることはないと思われます。  
リッチテキストでペーストされるのは止めたい、従来通りプレーンテキストだけコピーしたいという場合は、以下のように明示的に `plain` を設定します。

```
keybind = cmd+c=copy_to_clipboard:plain
```

`vt` は面白いオプションで、エスケープシーケンスごとコピーできます。ターミナル間でのコピペに有用…かも？

## スクロールバック検索

検索機能が追加されました。ほかの多くのターミナルアプリにはすでにあったものですね。  
内部的には専用の検索スレッドが使われており、メインの処理を妨げずに検索できるとのことです。

`cmd+f` で検索バーが開き、スクロールバック全体を検索できます。検索バーはドラッグして移動可能です。  
検索ヒットの前後移動は `cmd+g` / `shift+cmd+g` です。

![](https://www.youtube.com/watch?v=cGVIa2CdSSk)

ただ、ちょっとこの検索機能、直感と異なる点がいくつかあると感じています。

まず、どうやら検索窓にフォーカスが当たっている状態では `cmd+g` を押しても前後の検索位置へ移動できません。  
この前後移動はあくまでGhosttyのメインの画面内にフォーカスがある前提であり、検索中は無効のようです。  
いったん `tab` で検索窓からフォーカスを戻してやると、前後移動が効くようになります。

で、この前後移動は `next` が上方向へ、 `previous` が下方向へカーソルが進みます。今この記事を見ているブラウザの検索とは逆の挙動です。  
基本的にカーソルが一番下にあるというターミナルの特性上、上から下へ検索していったら確かに変なのですが、慣れた順序と逆だったので最初は少し戸惑いました。

## ネイティブスクロールバーの追加

ほかのGUIアプリケーションと同じようなスクロールバーが表示されるようになりました。そういえばこれまでなかったんですね。  
通常は非表示で、スクロールしたときのみ表示されます。

## コマンド完了通知

長時間実行するコマンドの完了を通知してくれる機能です。3つの設定で制御します。

![](https://storage.googleapis.com/zenn-user-upload/3b08d615f262-20260321.png)  
*通知結果の例*

### notify-on-command-finish

通知するかどうかの設定です。

- `never`: 通知しない（デフォルト）
- `always`: 常に通知（後述の時間設定があるので、すべてのコマンドで通知されるわけではない）
- `unfocused`: 実行サーフェスがフォーカスされていない場合のみ通知

### notify-on-command-finish-action

通知方法の設定です。

- `bell`: ベル（デフォルトで有効）
- `notify`: デスクトップ通知（デフォルトで無効）

`no-` をつけることで無効にでき、かつ組み合わせが可能です。デフォルトは `bell,no-notify` です。  
ベルの挙動は `bell-features` オプションで設定できます。  
macOSでは1.3.0から `bell-features = audio` が動作するようになったので、独自の音声ファイルを使用することも可能です。

### notify-on-command-finish-after

コマンド実行からこの時間が経過した場合のみ通知されます。デフォルトは5秒。  
これを設定することで、「長時間かかるコマンドのみ通知を出す」ことが可能になります。  
ただ、上限は `584y 49w 23h 34m 33s 709ms 551µs 615ns` なので、これ以上の時間が見込まれるコマンドを実行する場合は注意してください。

## キー設定の更新

### モーダルキーバインド

tmuxやVimのようなモーダルキーバインドを定義できる「Key Tables」が追加されました。  
これまでのキー設定の前に `{table-name}/` を前置します。

```
# "resize" テーブルを定義
keybind = resize/arrow_up=resize_split:up,10
keybind = resize/arrow_down=resize_split:down,10
keybind = resize/arrow_left=resize_split:left,10
keybind = resize/arrow_right=resize_split:right,10
keybind = resize/escape=deactivate_key_table
keybind = resize/catch_all=ignore

# ⌘⇧r でテーブルをアクティベート
keybind = super+shift+r=activate_key_table:resize
```

この例では `super+shift+r` を押すとresizeモードに入り、矢印キーだけでスプリットをリサイズできます。 `escape` で通常モードに戻ります。 `catch_all` は未定義のキーをすべてキャッチするワイルドカードで、このサンプルではモードに関係ないキーはすべて無効化しています。

今どのモードにいるかは画面下部に表示されるインジケーターで確認できます。

![](https://www.youtube.com/watch?v=MAA9E4B9uUg)

### chainキーバインド

`chain` を使えば、1つのキーに複数アクションを割り当てることもできます。これは本リリースに先駆けて記事化しているのでこちらをご覧ください。

### キーリマップ

`key-remap` 設定でGhostty内でのキーリマップが可能になりました。macOSだとkarabinerが有名ですが、これをGhostty内で設定できます。  
例えば `key-remap = ctrl=super` とすると、Ghostty内でCtrlがSuperとして扱われます。使い所がちょっとむずそうだけど便利かも。

## 作業ディレクトリ継承制御

新しいタブ、スプリット、ウィンドウを開いたときに、現在の作業ディレクトリを引き継ぐかどうかを個別に制御できるようになりました。

```
window-inherit-working-directory = false
tab-inherit-working-directory = true
split-inherit-working-directory = true
```

例えば「タブとスプリットは現在のディレクトリを引き継ぐけど、新規ウィンドウはホームディレクトリから始める」といった設定が可能です。

## AppleScriptサポート（プレビュー）

macOSでAppleScriptによるターミナル操作が可能になりました。ウィンドウ、タブ、スプリットの操作やテキスト入力を自動化できます。  
まぁ…いろいろ面白いことができそうですね。

```
tell application "Ghostty"
    set term to focused terminal of selected tab of front window
    input text "pwd\n" to term
end tell
```

macOSのオートメーション権限（TCC）で保護されているため、初回実行時に許可が必要です。 `macos-applescript = false` で機能を無効化できます。

なお、現段階ではプレビュー機能として提供されており、今後APIが変更される可能性があります。

## スプリットの移動・均等化

スプリットの上部付近にマウスを置くとグラブハンドルが表示され、ドラッグで並べ替えができるようになりました。別タブへの移動や新規ウィンドウへの切り出しも可能です。ターミナルの内容、作業ディレクトリ、実行中のプロセスがすべて一緒に移動します。Undo/Redoにも対応しています。

さらに、スプリットの境界線をダブルクリックするとスプリットが均等サイズにリセットされます。地味に便利。

![](https://www.youtube.com/watch?v=M41w6G6U6PU)

別タブに移動する際にはタブバーにカーソルをあわせて少し待つ必要があります。

## コマンドパレット拡張

`command-palette-entry` が1.3でmacOSでも使えるようになりました。実行したいアクションを事前に定義しておき、コマンドパレットから呼び出して実行できます。

また、コマンドパレットにセッション検索機能が追加されました。ターミナルのタイトルや作業ディレクトリで検索して、目的のサーフェスにジャンプできます。  
`Focus` で検索してもいいし、ディレクトリ名で直接検索することも可能。別ウィンドウや別タブ内の分割ペインみたいなところにもジャンプできるので便利です。  
ただ、試してみた限りでは（現状のコマンドパレットが全てそうですが）fuzzy findはできないようでした。

![](https://www.youtube.com/watch?v=IE5sVutVdlI)

### 設定ファイルの.ghostty 拡張子

Ghosttyの設定ファイルに`.ghostty` 拡張子が使えるようになりました。今までは `~/.config/ghostty/config` という拡張子のないファイルでした。

### Readonly サーフェス

toggle\_readonlyサーフェスを読み取り専用としてマークできるようになりました。  
デフォルトキーバインドはないので、自分で `toggle_readonly` アクションをマップするか、コマンドパレットから呼び出す必要があります。  
画面上での選択やコピー等はできますが、実行処理へは入力イベントが一切送信されなくなります。また、この状態のままサーフェスを閉じようとすると警告が表示されます。  
実行中のプロセスをミスって止めたりするのを防止できるので、サーバーログを流し続ける画面にしておくみたいな用途に良いのではないでしょうか。

### マウスレポーティングの無効化

`mouse-reporting = false` で、実装中プロセスにマウスイベントが届かなくなります。TUIアプリをマウスで誤操作するのを防ぐのにどうぞ。設定ファイルで指定するほか、キーバインドやコマンドパレットからトグルすることも可能です。

macOSのTerminal.appの機能としてあったらしいです。知らなかった。

### カスタムシェーダーの充実

カスタムシェーダーに、カーソル形状・位置・前回位置・変更からの経過時間・カラースキームなど、新しいuniformがいろいろ追加されたとのこと。遊びたい！！

## 開発体制の安定化（たぶん）

これはアプリケーションの更新ではありませんが、1.2から1.3の間に、Ghosttyは米国の非営利団体Hack Clubのスポンサーシップのもとで運営される非営利プロジェクトとなりました。正式に寄付も受け付けるようになっています。

知的財産はHack Clubに移管されたものの、これはOSSでなくなるというわけではなく、MITライセンスで開発していくことを維持。何人かのコアコントリビューターには正式に契約と報酬支払いを行ったとのことです。

また、Mitchell本人は1セントも受け取らないとのこと。詳しくはMitchellのブログ記事を参照してください。

## libghostty

これもGhostty自体の更新ではありませんが、1.3までの間にGhostty本体からlibghosttyの抽出が完了しました。  
ゼロからターミナルエミュレータを作らなくても、「何らかのアプリケーションにターミナル部分としてlibghosttyを利用する」ということができるようになっています。

時期は未定ですが、Ghosttyのアプリケーション部分とlibghosttyは独立してリリース管理されていく予定のようです。

## 更新しよう

更新内容いっぱいの1.3でした。  
機能は増えてはいますが、「細かく設定しなくてもちゃんと使える」というGhosttyの思想は続いているので、設定ファイルを書かなくてもマウスでクリックしたりコマンドパレットから探したりすればだいたい使えると思います。

紹介できていない項目もあるので、興味があればリリースノートを見てみてください。

あと翻訳追加PRを出したことで筆者もにほんごチームの一員となりました。わーい。  
まだmacOS向けには翻訳はリリースされていませんが。

今後はGhosttyだけでなくlibghosttyの発展も面白くなりそうですね。
